# build-deck v1.1 Motion Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a three-layer motion system to build-deck's Present mode: slide transitions (per-theme defaults + override), element entrance cascade, and per-theme background motion signatures. All CSS-only.

**Architecture:** Present-mode slide layout switches from `display: none/block` to a CSS grid stack so old + new slides can overlap during a transition. Transition variants apply via `data-transition` attribute on `<html>`. Entrance cascade uses CSS variables so Brutalist + Lab can opt out. Background motion lives on `body` or `.deck` to avoid colliding with slide-level transition animations; a JS-triggered `body.is-pulsing` class handles per-slide-change effects (Mono pulse).

**Tech Stack:** Vanilla HTML/CSS/JS. CSS `@keyframes`, `data-*` attributes, CSS variables. No new dependencies. Verification via headless browser (gstack `browse` skill).

**Reference:** [Design doc](./2026-05-19-v1.1-motion-design.md) · [Issue #5](https://github.com/LeoHChen/claude-skills/issues/5)

## Task 5: Update `showSlide` for entering/exiting class lifecycle

**Files:**
- Modify: `build-deck/template/index.html` (JS — `showSlide` and add `transitionTo`)

- [ ] **Step 1: Replace `showSlide`**

Find:
```js
function showSlide(idx) {
  const slides = getSlides();
  const total = slides.length;
  if (total === 0) return;
  currentIndex = Math.max(0, Math.min(total - 1, idx));
  slides.forEach((s, i) => s.classList.toggle("is-current", i === currentIndex));
  updateChrome();
}
```

Replace with:
```js
function showSlide(idx) {
  const slides = getSlides();
  const total = slides.length;
  if (total === 0) return;
  const newIdx = Math.max(0, Math.min(total - 1, idx));
  const oldSlide = slides[currentIndex];
  const newSlide = slides[newIdx];
  const animating =
    document.documentElement.dataset.mode === "present" &&
    oldSlide && oldSlide !== newSlide &&
    document.documentElement.dataset.transition !== "cut" &&
    !window.matchMedia("(prefers-reduced-motion: reduce)").matches;
  if (animating) {
    transitionTo(oldSlide, newSlide);
  } else {
    slides.forEach((s, i) => {
      s.classList.toggle("is-current", i === newIdx);
      s.classList.remove("is-entering", "is-exiting");
    });
  }
  currentIndex = newIdx;
  updateChrome();
}

function transitionTo(oldSlide, newSlide) {
  oldSlide.classList.remove("is-current");
  oldSlide.classList.add("is-exiting");
  const onOldEnd = () => {
    oldSlide.classList.remove("is-exiting");
    oldSlide.removeEventListener("animationend", onOldEnd);
  };
  oldSlide.addEventListener("animationend", onOldEnd);

  newSlide.classList.add("is-current", "is-entering");
  const onNewEnd = (e) => {
    if (e.target !== newSlide) return;
    newSlide.classList.remove("is-entering");
    newSlide.removeEventListener("animationend", onNewEnd);
  };
  newSlide.addEventListener("animationend", onNewEnd);
}
```

The `e.target !== newSlide` guard on `onNewEnd` is needed because animationend bubbles from child elements (the entrance cascade in Task 8). Without it, the first child cascade event would remove `.is-entering` prematurely.

```js
const transitionLabels = { auto: "Auto", fade: "Fade", "slide-h": "Slide", zoom: "Zoom", cut: "Cut" };
const motionTrigger = document.getElementById("motion-trigger");
const motionMenu = document.getElementById("motion-menu");
const motionCurrent = document.getElementById("motion-current");

function setTransitionPref(name) {
  localStorage.setItem("deck-transition", name);
  motionCurrent.textContent = transitionLabels[name] || name;
  motionMenu.querySelectorAll("[data-transition-name]").forEach(b => {
    b.setAttribute("aria-current", String(b.dataset.transitionName === name));
  });
  applyTransition();
}

function closeMotionMenu() {
  motionMenu.hidden = true;
  motionTrigger.setAttribute("aria-expanded", "false");
}

motionTrigger.addEventListener("click", () => {
  const willOpen = motionMenu.hidden;
  motionMenu.hidden = !willOpen;
  motionTrigger.setAttribute("aria-expanded", String(willOpen));
});

motionMenu.addEventListener("click", e => {
  const btn = e.target.closest("[data-transition-name]");
  if (btn) { setTransitionPref(btn.dataset.transitionName); closeMotionMenu(); motionTrigger.focus(); }
});

document.addEventListener("click", e => {
  if (!motionMenu.hidden && !motionTrigger.contains(e.target) && !motionMenu.contains(e.target)) {
    closeMotionMenu();
  }
});

document.addEventListener("keydown", e => {
  if (e.key === "Escape" && !motionMenu.hidden) { closeMotionMenu(); motionTrigger.focus(); }
});
```

```js
function transitionTo(oldSlide, newSlide) {
  // Layer C: theme-specific bg pulse triggers
  document.body.classList.remove("is-pulsing");
  void document.body.offsetWidth;
  document.body.classList.add("is-pulsing");
  setTimeout(() => document.body.classList.remove("is-pulsing"), 1200);

  oldSlide.classList.remove("is-current");
  oldSlide.classList.add("is-exiting");
  const onOldEnd = () => {
    oldSlide.classList.remove("is-exiting");
    oldSlide.removeEventListener("animationend", onOldEnd);
  };
  oldSlide.addEventListener("animationend", onOldEnd);

  newSlide.classList.add("is-current", "is-entering");
  const onNewEnd = (e) => {
    if (e.target !== newSlide) return;
    newSlide.classList.remove("is-entering");
    newSlide.removeEventListener("animationend", onNewEnd);
  };
  newSlide.addEventListener("animationend", onNewEnd);
}
```

```markdown
## Motion

In Present mode, build-deck plays three layers of motion. All CSS-only, gated by `prefers-reduced-motion`.

### Slide transitions

Topbar **Motion** selector lets the user pick between Auto (theme default), Fade, Slide, Zoom, or Cut. Per-theme defaults:

| Theme | Default | Theme | Default |
|---|---|---|---|
| Terracotta | Fade | Mono | Slide |
| Carbon | Slide | Aurora | Zoom |
| Berry | Fade | Brutalist | **Cut** (deliberate) |
| Lab | Slide | Dusk | Fade |

Selection persists to `localStorage.deck-transition`.

### Element entrance cascade

When a slide becomes current, child elements (title, subtitle, cards, pillars, flow nodes) stagger in over ~500ms total. Brutalist and Lab opt out — they're deliberately static.

### Background motion (per theme)

| Theme | Motion |
|---|---|
| Aurora | Gradient hue-drift, 20s loop |
| Mono | Dotted grid pulse on each slide change |
| Carbon | Scanline drift, 8s loop |
| Berry | Vignette breathe, 8s loop |
| Terracotta | Paper grain texture (static) |
| Dusk | None (deferred to v1.2 — needs JS particles) |
| Brutalist, Lab | None (deliberately static) |

All motion is suppressed when the OS reports `prefers-reduced-motion: reduce`.
```
