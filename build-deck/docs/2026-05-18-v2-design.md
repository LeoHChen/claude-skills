# build-deck v2 — Design Doc

**Date:** 2026-05-18
**Status:** Approved for implementation planning
**Scope:** `claude-skills/build-deck` skill

## Goal

Upgrade build-deck from a scroll-only deck to a dual-mode (Review / Present) deck with a wider set of modern themes, native presentation chrome (progress hairline, floating nav, page numbers), and a fix for a latent theme-CSS bug. The output stays self-contained HTML + content.md and PDF export keeps working.

## Motivation

Today the rendered deck is one long vertical scroll of every slide. That's good for drafting and review, but useless for actually presenting. No keyboard nav, no progress indication, no page numbers. The user has four themes (Terracotta, Carbon, Berry, Lab); modern decks would benefit from more aesthetic range without losing the "single source of truth" YAML-content pattern.

## Non-goals

The following are deliberately out of scope:
- Speaker notes / presenter view
- Slide thumbnails / overview grid
- Transition animations between slides
- Mobile swipe gestures
- Per-slide theme overrides
- A schema change to `content.md`

None of these block this change; they can be added later if needed.

## Design

### Modes

Two modes, toggled in the topbar and persisted to `localStorage.deck-mode`:

- **Review** (default for editing) — current scroll layout, all slides stacked vertically. Topbar shows: Mode toggle, Theme dropdown, Export PDF.
- **Present** — one slide fills the viewport. Adds progress hairline, floating nav bar, and visible on-slide page numbers. Esc returns to Review.

### Navigation in Present mode

- **Keys**: `← / →` (prev/next), `Space` (next), `Home / End` (first/last), `Esc` (exit to Review).
- **Floating bar** at bottom-center: `‹ Prev   03 / 12   Next ›`. Appears on `mousemove`, fades after 2s of idle. Theme-aware (uses `--card` + `backdrop-filter`).
- No click-to-advance — avoids conflicts with any inline links in future slide types.

### Progress hairline

- 2px line, fixed to top of viewport, **Present mode only**.
- Width = `(currentIndex + 1) / total * 100%`. Color `--primary`. Transition 200ms ease.
- Hidden in Review and in print output.

### Page numbers

- Bottom-right corner of every slide, both modes.
- Format: zero-padded mono, e.g. `03 / 12`. Color `--text-subtle`.
- Lives in the slide DOM (not in nav chrome) so it survives `window.print()` with no extra print CSS.

### Themes

The existing four themes are unchanged. Four new themes are added:

| Theme | Bg | Primary | Accent | Display | Notes |
|---|---|---|---|---|---|
| **Mono** | `#FFFFFF` | `#00B8D4` cyan | `#FF3366` magenta-red | sans, tight tracking | radius 8px, hairline borders |
| **Aurora** | `linear-gradient(135deg, #2A1B4A, #1A3D52)` | `#A78BFA` lavender | `#5EEAD4` mint | sans display | glass cards via `backdrop-filter: blur(20px)` |
| **Brutalist** | `#FFFFFF` | `#CCFF00` chartreuse | `#000000` | heavy sans-black | radius 0, no shadows, 2px hard rules |
| **Dusk** | `#1A1422` | `#C4B5FD` lavender | `#FCA5A5` peach | serif | premium dark, no navy |

With eight themes, the topbar chip row is replaced by a **Theme dropdown**: a single button labeled with the current theme, click opens a menu of all eight. State persists to `localStorage.deck-theme` (existing key, no migration needed).

### Bug fix

In `template/index.html` line 31, the selector `[data-theme="terracotta"]` is duplicated where `[data-theme="carbon"]` was intended. The Carbon token block silently overrides Terracotta, so the "Terracotta" button applies Carbon's dark palette. One-character selector fix ships with this work; verified by manually toggling each theme after the change.

### PDF export

- Print CSS unchanged in Review mode.
- In Present mode, **Export PDF** temporarily switches layout to Review for the print job, then restores. Implemented by toggling `data-mode="review"` on `<html>` around the `window.print()` call.
- Page numbers print correctly because they're in the slide DOM, not the nav chrome.
- Aurora's gradient + glass-card backgrounds rely on `print-color-adjust: exact`, which is already set globally.

### State summary

| Key | Values | Default |
|---|---|---|
| `localStorage.deck-theme` | one of 8 theme names | `terracotta` (existing) |
| `localStorage.deck-mode` | `review` \| `present` | `review` (new) |

## Risks & open questions

- **Aurora print fidelity.** Gradient backgrounds and glass-card `backdrop-filter` may render differently across Chrome/Safari/Firefox print pipelines. Acceptance: Chrome `Save as PDF` matches the on-screen rendering pixel-close. Other browsers are best-effort.
- **Dropdown a11y.** Needs `aria-haspopup`, `aria-expanded`, focus trap on open, Esc-to-close. Standard, but worth calling out so it lands in the implementation.
- **Brutalist + Aurora are stylistic opposites.** Some slide types (e.g. `flow` with its dashed connector) may need theme-specific tweaks to look intentional rather than broken. Acceptance: visually inspect every slide type in every theme before merge.

## Acceptance criteria

1. Toggling Mode in the topbar switches between Review (scroll) and Present (single-slide) layouts; state persists across reload.
2. In Present mode: `← / → / Space / Home / End / Esc` keys behave as specified.
3. Progress hairline reflects current position; fills smoothly on advance; absent in Review and print.
4. Page number `NN / MM` is visible on every slide in both modes and in the exported PDF.
5. Theme dropdown lists all 8 themes; selecting any one applies it instantly and persists.
6. Selecting "Terracotta" produces the warm-cream palette (not Carbon's dark one).
7. Export PDF in Present mode produces a PDF with the same content and layout as Export PDF in Review mode (one slide per page, 16:9 at 13.333in × 7.5in, page numbers visible, all themes).
8. All four new themes render every slide type (`title`, `architecture`, `cards`, `flow`, `roadmap`, `questions`) without overflow or broken layout.
